API ENDPOINTS EXPLAINED

Will Be Using Examples From Character Controller As It Has A Few Unique Endpoints

Some end-points listed are equal, however several methods may be applied so therefore will be listed more than once

Endpoint 1: Get (https://localhost:7127/api/Characters)

Description: This endpoint retrives a list of JSON elements

Endpoint 2: Post (https://localhost:7127/api/Characters)

Description: This endpoint recieves a post request, it accepts a key/value pair of FormData aswell as a JSON object

Endpoint 3: Get (https://localhost:7127/api/Characters/{id})

Description: This endpoint takes the object id of the element as an argument to find one of equal match, it retrieves a list of JSON elements

Endpoint 4: Put (https://localhost:7127/api/Characters/{id})

Description: This endpoint accepts a JSON element in a PUT request, it takes the id of an existing element to change make changes to its values

Endpoint 5: Delete(https://localhost:7127/api/Characters/{id})

Description: This endpoints accepts the id of an element as argument for which element to remove from the database

Endpoint 6: Get(https://localhost:7127/api/Characters/name/{query})

Description: This endpoint takes an argument and attempts to match the argument supplied with any of the attributes stored in the database, returns a list of matches

Endpoint 7: Get(https://localhost:7127/api/Characters/randomizefour)

Description: This endpoint retrieves a list of 4 random elements by using the Random() method

Endpoint 8: Post(https://localhost:7127/api/file)

Description: This endpoint accepts 2 arguments: a string which is used to name the file, including the FormFile in binary for the data of the file
